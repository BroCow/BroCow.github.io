The Magic of Media Queries

What is a Media Query?

Media queries are used to “query,” or check for what type of device or condition the user is utilizing to view a webpage.

They can check for things such as:

  • Width and height of the viewport
  • Width and height of the device
  • Orientation (is the tablet or phone in landscape or portrait mode?)
  • Resolution

When should they be used?

Media queries should be used whenever the webpage layout or style should change depending on what device is being used to view it.

Why are they helpful?

They are very helpful because they provide an improved user experience by customizing the function and appearance of a page for specific viewports.

Where are they placed?

CSS

/*At the top level of your code */
@media only screen and (min-width = 37.5rem)
{
article {
Padding: 1rem 3rem;
}
}

How are they coded?

Always design for mobile first. This means that the design is made to work first on a mobile device before tablet or desktop, which helps to ensure that the page will load at good speeds on smaller devices.

This means that as we change the webpage, we make adjustments as the page gets larger, not as it gets smaller.

Media Query Syntax Breakdown

@media [not|only] [media type] [and] ([media feature and|or|not media feature])
{
CSS-code;
}

Example of an At Rule. These begin with the @sign.

  • Media
    • A conditional group rule that will apply its content if the device meets the criteria of the condition defined when media query is used
  • Not, Only, And
    • Not> - Reverts the meaning of an entire media query
    • Only> - Prevents older browsers that do not support media queries with media features from applying the specified styles. It has no effect on modern browsers.
    • And> - Combines media feature with a media type or other media feature
  • Media Type
    • All> - Default…used for all media type devices
    • Print> - Used for printers
    • Screen> - Used for computer screens, tablets, smart-phones
    • Speech> - Used for screen-readers that read the page out loud
  • Media Feature
    • Orientation
    • Width
    • Height
    • Resolution
    • Other Examples

Media Query Break Points

There are so many devices out there with various widths and heights, so coding a media query for every single one is not realistic. However, there are five typical groups that should be targeted as listed below:

  • Extra Small Devices (phones, 600px and down)
    @media only screen and (max-width: 600px)

  • Devices (portrait tablets and large phones, 600px and up)
    @media only screen and (min-width: 600px)

  • Medium Devices (landscape tablets, 768px and up)
    @media only screen and (min-width: 768px)

  • Large Devices (laptops and desktops, 992px and up)
    @media only screen and (min-width: 992px)

  • Extra Large Devices (large laptops and desktops, 1200px and up)
    @media only screen and (min-width: 1200px)
